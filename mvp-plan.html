03 · Roles & Permissions

Seven roles. Two sides of the wall.

The architecture is tenant-first, school-second. A tenant (e.g. Pflegeakademie Datteln) can run multiple schools. School Admins manage their school; Tenant Directors manage every school under the tenant.

LingoCare's three internal roles run the platform: Super Admin (us, full control), Admin (most operations), Sales (read-only analytics).

The tenant side has four user-facing roles. Each invites only downward: Tenant Director → School Admin → Instructor → Learner.

Platform: CASL with tenant_id + school_id enforced at the model layer. Full permission matrix in the RBAC doc.

07 · Scope Pressure

If we slip, we cut in this order.

Relief valves

Cut from the top. Each lower number means more painful but still survivable.

  • 01Announcement system → email instead
  • 02Director CSV export → screenshot
  • 03LLM re-ranker → vector + metadata sufficient
  • 04Language Adapter → generate B1 only, adapt manually
  • 05PF3 simulation types → ship PF1 + PF2 only
  • 06Gamification (if W4 says go) → no-go = no change
  • 07Pronunciation STT → text-based listening only
  • 08Adaptive referral → learners find Garden manually
  • 09Lingo for instructor + director → learner only
